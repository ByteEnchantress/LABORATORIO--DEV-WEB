Jakarta Server Pages (JSP)

Jakarta Server Pages (JSP): es un lenguaje de scripting del lado del servidor que permite la creación de aplicaciones web dinámicas. JSP se basa en Java y tiene acceso completo a las API de Java, incluyendo JDBC para interactuar con bases de datos.

Elementos de JSP

Scriptlet

Un scriptlet en JSP permite incluir fragmentos de código Java en una página JSP. Se puede usar para ejecutar cualquier código Java válido.

<%
out.println("Hola desde JSP");
%>

Declaraciones

Una declaración en JSP se usa para definir variables o métodos que se utilizarán en el resto del archivo JSP. Las declaraciones deben hacerse antes de usarlas.

<%!
int i = 0;
int a, b, c;
Circle a = new Circle(2.0);
%>

Expresiones

Una expresión en JSP se evalúa y se convierte en HTML en la página resultante. Se utiliza para insertar dinámicamente datos en el contenido HTML.

<html>
<head><title>A Comment Test</title></head>
<body>
<p>Today's date:
<%= (new java.util.Date()).toLocaleString()%>
</p>
</body>
</html>

¿Qué aprendí sobre JSP?

Aprendí que los scriptlets permiten la inserción directa de lógica Java en el código HTML, facilitando la creación de contenido dinámico en la web. Ademas, las declaraciones son útiles para definir variables y métodos que se mantendrán durante el ciclo de vida de la página JSP, lo que permite un código más organizado y reutilizable. Complementado con que las expresiones en JSP son una forma poderosa de generar contenido dinámico y de actualizar la información en tiempo real, lo que mejora la interacción con el usuario.

¿Cómo aprendí?

Aprendí JSP mediante la creación de ejemplos prácticos y la revisión de la documentación oficial. Implementé y probé los conceptos básicos como scriptlets, declaraciones y expresiones en proyectos reales, lo que me permitió entender cómo integrarlos en páginas web dinámicas.
