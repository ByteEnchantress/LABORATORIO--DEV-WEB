Desarrollar Aplicaciones Dinámicas con Next.js

Next.js es un framework fullstack JavaScript que utiliza React para crear componentes. Gracias a los "React Server Components (RSC)", Next.js permite tener componentes tanto en el servidor como en el cliente, ofreciendo flexibilidad en el método de renderizado.

Rendering y Server-Side Rendering

El renderizado es el proceso de convertir código JavaScript y CSS en HTML. Este proceso puede ocurrir en el servidor (Server-Side Rendering) o en el cliente (Client-Side Rendering). Next.js soporta tres métodos de renderizado:

  • Server-Side Rendering (SSR): Renderizado de la aplicación en el servidor y envío al cliente.
  • Static Site Generation (SSG): Generación de sitios estáticos en tiempo de compilación.
  • Client-Side Rendering (CSR): Renderizado en el cliente después de que la aplicación se carga.

Componentes de Servidor y Componentes de Cliente

Con los componentes de servidor, se puede aprovechar mejor la infraestructura del servidor, mover la obtención de datos más cerca de la base de datos y mantener dependencias grandes en el servidor, mejorando el rendimiento.

Los componentes del cliente se especifican con la directiva "use client" y se renderizan en el cliente.

Typescript

TypeScript es un superconjunto de JavaScript que añade tipado estático y objetos basados en clases, permitiendo declarar variables y objetos con tipos específicos.

Ejemplo de TypeScript

// Dato de tipo string var name: string = 'Tu nombre'; name = 'Otro nombre'; // Es correcto name = 2 // Es incorrecto // Dato de tipo number var age: number = 29; age = 0xf00d; // Es hexadecimal y es correcto age = '3'; // Es un string e incorrecto // Dato de tipo boolean var havePets: boolean = true; havePets = false; // Es correcto havePets = 3 // Es incorrecto // Ejemplo de Interface interface Puppy { name: string, age: number }; const puppy: Puppy = { name: "Mascota", age: 2 }; // Ejemplo de Arreglos let arrayNumber: number[] = [1, 2, 3]; let arrayString: string[] = ['1', '2', '3'];
